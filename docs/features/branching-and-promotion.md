# Branching and Promotion

OriCMS uses branch-aware content workflows so teams can stage, review, and promote changes without forcing everything through one live editing stream.

## What The Feature Does

- project content is branch-aware
- promotion and merge flows are serialized
- branch-aware locks and revisions prevent conflicting high-impact changes

In practice, this gives teams a place to do review work before a change reaches the branch that drives a production environment.
