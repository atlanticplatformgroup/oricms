# Schemas and Content Types

Schemas, content types, and components define what content can exist in a project and how editors work with it.

This is one of the most important feature layers in OriCMS. If the schema is clean, the editing experience tends to feel clear. If the schema is muddy, everything downstream gets harder.

## What The Feature Does

- content types define top-level entry models and display behavior
- component schemas define reusable nested structures
- field behavior is shared across web and API through shared contracts

## Typical Workflows

- create a content type
- create a component schema
- edit fields and display config
- delete or replace a schema safely
- select components from content-type fields in the builder
- use components and blocks to model rich content
