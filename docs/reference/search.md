# Search

Use this page when you need the search model itself rather than broader product advice.

## Built-In Search

OriCMS provides built-in collection search driven by shared field capability logic.

This is useful for:

- collection browse filtering
- lightweight editorial lookup
- basic headless content search over modeled content

## What It Is Not

The current built-in search is not a full production search engine.

It does not attempt to be a replacement for:

- typo-tolerant search
- advanced relevance ranking
- faceting
- click analytics
- large-scale public search indexing

## When External Indexing Makes Sense

Use external indexing when you need production-grade site search behavior, especially across many collections or rendered site pages.

The cleanest mental model is simple: OriCMS can answer content queries, but it should not be forced to act like a dedicated search product once search becomes a feature in its own right.
