# Self-Hosting

Use this page for self-hosted OriCMS deployments.

## Current Needs

A self-hosted deployment typically needs:

- API runtime
- web admin app
- database
- repository workspace storage
- any supporting services used by the deployment

## Operational Focus

- keep project repositories and database storage healthy
- manage secrets and auth configuration carefully
- verify publish hooks and delivery targets after deployment changes
