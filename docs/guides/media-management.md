# Media Management

Use this guide when you need to upload assets, add metadata, and reuse those files across entries.

Media work in OriCMS is straightforward once you separate two concerns:

- the binary file itself
- the metadata that helps editors find and use that file well

You are usually doing three things in the Media workspace:

1. Upload the asset you need.
2. Add the metadata that makes it reusable.
3. Insert it from an image or media field when editing entries.

## A Good Working Routine

For most teams, the cleanest routine looks like this:

1. Upload the original asset from the **Media** workspace.
2. Add the basics immediately: alt text, caption, and any tags your team actually uses.
3. Reuse the same asset from entries instead of uploading duplicates.
4. Before deleting anything, check whether the asset is already referenced elsewhere.

That last step matters. Media libraries get messy when teams treat them like temporary upload buckets.

## What Metadata Is For

Metadata is what turns a file into something the rest of the project can work with.

Use it to:

- make assets searchable
- capture accessibility text such as alt text
- give editors a human-readable caption or description
- organize reuse with tags

The binary file and the metadata are related, but they are not the same thing. In practice that means you can improve metadata without re-uploading the asset.

## How To Keep The Library Usable

If you want the media library to stay usable after the first few weeks, a little discipline goes a long way:

- prefer reusing existing assets over uploading a slightly different duplicate
- use tags consistently instead of inventing one-off labels
- add alt text when the asset will appear in public content
- delete cautiously, especially for files used across multiple entries

Tags are the main organizational primitive today. Older folder-style metadata should be treated as compatibility baggage, not the center of your workflow.
