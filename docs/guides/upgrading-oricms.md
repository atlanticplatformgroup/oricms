# Upgrading OriCMS

Use this guide when moving a deployment or workspace to a newer OriCMS version.

## Current Guidance

- review decision and support-policy docs before major changes
- review migrations and compatibility notes
- verify package and adapter behavior in local or staging environments before production rollout
- treat curated compatibility paths as temporary bridges, not upgrade targets

## Upgrade Checklist

1. read the support and compatibility policies
2. review migration and schema changes
3. run verification in the workspace
4. test adapter and publish-loop behavior if applicable
5. roll out production changes after staging validation
